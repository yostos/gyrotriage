# ADR-002: 単一ファイル入力の原則

## ステータス

承認済み

## コンテキスト

gyrotraigeのCLIインターフェースにおいて、入力としてディレクトリ（フォルダ）を受け付けて再帰的にMP4ファイルを探索・バッチ処理する機能を持つべきかどうかを検討した。

## 決定

**gyrotraigeは常に単一のMP4ファイルのみを入力として受け付ける。** フォルダの再帰探索機能およびバッチ処理機能は実装しない。これはMVPの制約ではなく、永続的な設計原則である。

## 根拠

UNIX哲学「1つのことをうまくやる（Do One Thing and Do It Well）」に基づく。

- gyrotraigeの責務は「1つのMP4ファイルを解析してブレスコアを出力する」ことに限定する
- 複数ファイルの処理はシェルに委ねる。これにより、フィルタリング・並列実行・出力の加工など、ユーザーが自由に組み合わせられる

```bash
# 全MP4を処理
for f in *.MP4; do gyrotriage "$f"; done

# 並列処理
ls *.MP4 | xargs -P4 -I{} gyrotriage "{}"

# 結果をファイルに保存
for f in *.MP4; do gyrotriage "$f" >> results.txt; done
```

ディレクトリ探索をツール内部に持つと、再帰の深さ、除外パターン、並列度の制御など、本来の解析とは無関係な機能の実装・テスト・保守が必要になる。これらはシェルとOSのユーティリティがすでに十分に提供している。

## 影響

- CLIの`<FILE>`引数は常に単一ファイルパス
- `--recursive`や`--glob`のようなオプションは存在しない
- バッチ処理の出力フォーマット（一覧表、ソートなど）は不要
