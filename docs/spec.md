# gyrotriage 機能仕様書

## 概要

DJI FPVドローンが撮影したMP4ファイルからモーションデータ（クォータニオン）を抽出・解析し、以下を提供するCLIツール。

1. 映像のブレ度合いのスコアリングと判定
2. Gyroflowスタビライゼーションの推奨パラメータ（ベースライン値）の提示

ユーザーはこの出力をもとにGyroflowのパラメータ調整の出発点とする。

## マイルストーン

### MVP

単一MP4の解析とテキスト出力。

| ID | ストーリー |
|---|---|
| US-1 | 単一MP4を指定してブレスコア・判定レベル・Gyroflow推奨パラメータをテキストで確認できる |
| US-2 | モーションデータのないファイルを渡した場合に原因がわかる |

### v2: --visual

| ID | ストーリー |
|---|---|
| US-3 | `--visual`でターミナルにブレの時系列グラフを表示できる |

### v3: --html

| ID | ストーリー |
|---|---|
| US-4 | `--html`でブラウザ閲覧可能なレポートを生成できる |

## 対象ファイル

### 対応機種

| 機種 | モーションデータ記録条件 |
|---|---|
| DJI Avata / Avata 2 | EIS（Rocksteady）オフ、FOV Wide |
| DJI Neo / Neo2 | アスペクト比 4:3（EIS自動オフ） |

### ファイル形式

- MP4コンテナ（QuickTime / MOV互換）
- DJI独自のprotobuf形式でモーションデータ（クォータニオン）がデータトラック（codec_tag: `djmd`）に格納されている

## CLIインターフェース

### 基本構文

```
gyrotriage <FILE> [OPTIONS]
```

`<FILE>` には単一のMP4ファイルを指定する。複数ファイルの処理はシェル側で行う（例: `for f in *.MP4; do gyrotriage "$f"; done`）。

### 引数

| 引数 | 説明 |
|---|---|
| `<FILE>` | 解析対象のMP4ファイル（必須） |

### オプション

| オプション | 説明 | マイルストーン |
|---|---|---|
| `--visual` | Sixel/iTerm2プロトコルによるグラフをターミナルに表示 | v2 |
| `--output-image <PATH>` | PNG画像ファイルに書き出し（ターミナル表示なし） | v2 |
| `--sparkline` | ANSIスパークラインをテキスト出力に追加 | v2 |
| `--sixel` | Sixelプロトコルを強制（`--visual` と併用） | v2 |
| `--iterm2` | iTerm2プロトコルを強制（`--visual` と併用） | v2 |
| `--html` | HTMLレポートファイルを生成 | v3 |

`--visual` と `--output-image` は併用可能。`--visual` と `--sparkline` は排他（`--visual` 優先）。

## 解析処理

### 入力データ

telemetry-parserクレートを使用し、MP4ファイルのデータトラック（`djmd`）からクォータニオンの時系列データを抽出する。

クォータニオンはDJI機体内で算出済みのカメラ3D姿勢であり、IMU生データ（角速度）ではない。

### 解析ステップ

1. MP4からクォータニオン時系列を抽出する
2. 連続フレーム間のクォータニオン差分から角変位量（ラジアン）を算出する
3. 角変位量をピッチ（X軸）・ロール（Z軸）・ヨー（Y軸）の3軸に分解する
4. 各軸および合成値について統計量を算出する
   - 平均角変位量
   - 標準偏差
   - 最大角変位量
   - ピーク頻度（閾値超過フレーム数）
5. 統計量からブレスコア（0〜100）を算出する
6. ブレスコアに基づいて判定レベルを決定する
7. ブレスコアおよび統計量からGyroflow推奨パラメータを算出する

### スコア算出式

1. 連続フレーム間クォータニオン差分: `q_diff = conj(q_t) × q_{t+1}`
2. 角変位量: `θ = 2 × arccos(min(|q_diff.w|, 1.0))`（ラジアン）
3. 角速度: `ω = θ（°変換後）/ Δt`（°/s）
4. 総合指標: 全フレームのRMS角速度
5. スコア: `score = min(100, rms_velocity / 20.0 × 100)`
   - 基準値 20°/s = スコア100（重度ブレ相当）

MVPではパン・チルトとブレの区別は行わない。全角変位をスコアに含める。

### 閾値テーブル

| レベル | スコア | RMS角速度 | 意味 |
|---|---|---|---|
| STABLE | 0–25 | < 5°/s | ブレがほぼなく、補正不要 |
| MILD | 26–50 | 5–10°/s | 軽微なブレあり、補正は任意 |
| MODERATE | 51–75 | 10–15°/s | 明確なブレあり、Gyroflow推奨 |
| SEVERE | 76–100 | > 15°/s | 激しいブレ、Gyroflow強く推奨 |

### Gyroflow推奨パラメータ

ブレの解析結果に基づき、Gyroflowで使用するパラメータのベースライン値を提示する。これはあくまで調整の出発点であり、最終的なパラメータはGyroflowのプレビューで映像を見ながらユーザーが決定する。

**smoothnessマッピング（区分線形）:**

| スコア | smoothness |
|---|---|
| 0–25 | 0.2 |
| 26–50 | 0.3〜0.5 |
| 51–75 | 0.5〜0.8 |
| 76–100 | 0.8〜1.5 |

**crop推定:**

```
crop ≈ 1 + smoothness × RMS角速度 / FOV°
```

RMS角速度 × smoothnessで安定化ウィンドウにおける実効角変位を推定し、FOVで割ってクロップ率を算出する。

FOV: DJI Neo 117.6°（機種固定値）

## 出力仕様

### テキスト出力（MVP）

```
File:        DJI_20260227_0001.MP4
Duration:    120.5s (24100 samples @ 200Hz)
Score:       72 / 100
Level:       MODERATE
RMS:         14.4 °/s
Peak:        45.2 °/s
Pitch:       avg=8.2°/s  std=4.5°/s  max=32.1°/s
Roll:        avg=3.4°/s  std=2.2°/s  max=18.7°/s
Yaw:         avg=5.6°/s  std=3.1°/s  max=24.4°/s
---
Gyroflow:    smoothness=0.5  crop≈1.2x
```

### モーションデータなしの場合（MVP）

```
File:        DJI_20260227_0005.MP4
Status:      NO MOTION DATA
Hint:        Neo/Neo2は4:3で撮影が必要。Avata/Avata2はEISオフ・FOV Wideが必要。
```

### グラフィカル出力 --visual / --output-image（v2）

詳細仕様は `docs/adr-004-visual-output.md` を参照。

1440×900px固定、ダーク背景のPNG画像を生成する。HUDスタイルの2カラムレイアウト:

- **左カラム（約1/3）**: SCOREゲージ（グラデーション弧）+ AXIS ANALYSISレーダーチャート
- **右カラム（約2/3）**: Composite / Pitch / Roll / Yaw の4折れ線グラフ（ライン+グロー、時間ラベルなし）
- **フッター**: MODERATE以上の場合のみGyroflow推奨パラメータを表示

配色はTokyo Night系・寒色基調（シアン/青/ティール/紫）。スコアゲージは緑→シアン→青→紫→赤のグラデーション。

`--visual`: Sixel/iTerm2プロトコルでターミナル横幅一杯に表示（8:5のアスペクト比を維持してスケーリング）。iTerm2は`width=100%`で自動フィット。Sixelは`ioctl(TIOCGWINSZ)`でターミナルのピクセル幅を取得しリサイズ。プロトコルは環境変数から自動検出（`--sixel`/`--iterm2`で上書き可能）。

`--output-image <PATH>`: PNGファイルに書き出し。`--visual`と併用可能。

### スパークライン --sparkline（v2）

テキスト出力の末尾にANSIブロック文字（▁▂▃▄▅▆▇█）によるスパークラインを追加する。SSHやパイプ環境での簡易可視化用。`--visual`と排他（`--visual`優先）。

### HTMLレポート --html（v3）

ブラウザで閲覧可能なHTMLファイルを生成する。

- インタラクティブなグラフ（マウスオーバーで値表示）
- 出力先: 入力ファイルと同じディレクトリに `<filename>_report.html` として生成
