# Gyroflow推奨パラメータ算出アルゴリズム

## 概要

gyrotraigeはMP4から抽出したクォータニオン時系列データをFFT/PSD（パワースペクトル密度）で周波数解析し、Gyroflowの5つのスタビライゼーションパラメータの推奨値を算出する。

これらの値はGyroflowでのパラメータ調整の出発点（ベースライン）であり、最終的なパラメータはGyroflowのプレビューで映像を確認しながらユーザーが決定する。

## 出力パラメータ一覧

| パラメータ | 単位 | Gyroflow JSONキー | 推奨範囲 |
|---|---|---|---|
| Smoothness | % | `smoothing_params[0].value` | 15–50% |
| Max smoothness | 秒 | `smoothing_params[1].value` | 0.3–2.0s |
| Max smoothness at high velocity | 秒 | `smoothing_params[2].value` | 0.03–0.3s |
| Zoom limit | % | `max_zoom` | 105–140% |
| Zooming speed | 秒 | `adaptive_zoom_window` | 2.0–6.0s |

## 入力データ

角速度時系列（Pitch/Roll/Yaw、単位: °/s）を`analyze.rs`で算出済み。元データはDJI MP4内のクォータニオン姿勢データから連続フレーム間の差分回転をオイラー角に分解したもの。

## 算出パイプライン

```
MP4 → クォータニオン抽出 → 角速度時系列 → FFT/PSD → カットオフ周波数推定 → パラメータ変換
```

### ステップ1: PSD（パワースペクトル密度）計算

角速度時系列の3軸合成値（RSS: Root Sum Square）に対してFFTを実行する。

1. Hann窓を適用してスペクトルリーケージを抑制
2. `rustfft`クレートでFFTを実行
3. 片側PSDを計算: `PSD[k] = |X[k]|^2 / (N × fs) × 2`

**なぜPSDを使うか**: 角速度信号は「意図した動き（低周波）」と「ブレ/振動（高周波）」の重ね合わせである。この2つは周波数帯域が明確に分離しており、PSDでその境界を客観的に検出できる。

FPVドローンの典型的な周波数帯域:

| 帯域 | 周波数 | 発生源 |
|---|---|---|
| 意図した動き | < 1 Hz | パン・チルト・旋回 |
| 手ブレ/風 | 3–10 Hz | 機体の姿勢変動 |
| モーター振動 | 20–80 Hz | プロペラ回転 |

### ステップ2: カットオフ周波数の推定

PSDから2種類のカットオフ周波数を推定する。

#### 主カットオフ周波数 fc（意図動作 vs ブレの境界）

0.5–5 Hzの範囲でPSD（移動平均平滑化済み）の極小値（谷）を探索する。この谷は意図した動きの周波数帯とブレの周波数帯の間に現れる。

**なぜ谷検出か**: ローパスフィルタのカットオフは「残すべき信号と除去すべき信号の境界」に置く必要がある。PSDの谷はまさにこの境界を示す。

最小値: 0.3 Hz（これ以下では有意な安定化ができない）

#### 高速回転カットオフ周波数 fc_hv

累積パワーの80%点を使用し、`fc * 2` 以上かつ10 Hz以下にクランプ。高速旋回中は低周波成分が支配的になるため、より高い周波数まで通過させる必要がある。

### ステップ3: 時定数への変換

カットオフ周波数からローパスフィルタの時定数を計算する:

```
τ = 1 / (2π × fc)
```

**なぜこの式か**: Gyroflowのスムージングは指数平滑フィルタ（EMA）を使用しており、その時定数τとカットオフ周波数fcの関係は `τ = 1/(2πfc)` で与えられる。Gyroflow内部の実装:

```
alpha = 1 - exp(-(1 / sample_rate) / smoothness)
```

ここで`smoothness`がτに相当する。

### ステップ4: 各パラメータの算出

#### Smoothness (%)

PSDから算出した**shake power ratio**（カットオフ以上の周波数帯のパワーが全体に占める割合）とRMS角速度から決定する。

```
base = 15 + 35 × shake_power_ratio
velocity_factor = 0.85 (rms < 3°/s) ~ 1.15 (rms > 15°/s)
smoothness = clamp(base × velocity_factor, 15, 50)
```

**なぜshake power ratioか**: 信号のうちブレが占める割合が大きいほど、より強いスムージングが必要になる。shake_power_ratio = 0なら信号は全て意図した動きであり最低限のスムージングで十分。shake_power_ratio = 1なら全てブレであり最大のスムージングが必要。

**なぜvelocity_factorか**: 同じshake ratioでも、RMS角速度が小さい（全体的に動きが少ない）場合は弱めのスムージングで十分であり、RMS角速度が大きい場合はより強いスムージングが効果的。

**FPVでの推奨範囲（20–35%）との整合**: shake_power_ratioが0.15–0.60の範囲で20–35%に収まる。FPVの典型的なフライトデータはこの範囲に入る。

#### Max smoothness（秒）

主カットオフ周波数fcから算出した時定数τをそのまま使用する。

```
max_smoothness = clamp(τ, 0.3, 2.0)
```

**なぜτをそのまま使うか**: Max smoothnessは「最大でどこまでスムージングするか」の上限。カットオフ周波数から求めた時定数はまさに「ブレを除去するために必要な最大フィルタ強度」に対応する。

**クランプ範囲の根拠**: Gyroflowの実用範囲。0.3s未満ではほぼ効果がなく、2.0s超ではクロップが過大になる。

#### Max smoothness at high velocity（秒）

高速回転カットオフfc_hvから算出した時定数を使用する。

```
max_smoothness_at_high_velocity = clamp(τ_hv, 0.03, 0.3)
```

**なぜ短い時定数か**: FPVの高速旋回時にスムージングが強すぎると「ラバー感」（映像が引きずられる不自然な動き）が発生する。高速回転時は意図した動きが支配的であるため、ブレ除去よりも追従性を優先する。

**クランプ範囲の根拠**: Gyroflowのデフォルトは0.1s。0.03s未満は事実上スムージングなし、0.3s超では高速旋回時の追従性が失われる。

#### Zoom limit (%)

SmoothnessとRMS角速度から推定する。

```
base = 105 + (smoothness_pct - 15) × 25 / 35
velocity_extra = min(rms_velocity / 20 × 5, 10)
zoom_limit = clamp(base + velocity_extra, 105, 140)
```

**なぜsmoothnessとRMS角速度の組み合わせか**: スムージングが強いほど、フレーム間の補正量が大きくなり、黒縁を隠すために必要なズーム量が増加する。また、RMS角速度が大きい（ブレが大きい）映像ではフレームごとの補正量の最大値も大きくなるため、追加のズーム余裕が必要になる。

**FPVでの目安（110–125%）との整合**: smoothness 20–35%、RMS 5–15°/sの典型的なFPVデータで110–120%に収まる。

#### Zooming speed（秒）

角速度の**時間的変動性**（ローリングRMSの変動係数CV）から算出する。

1. 3軸合成角速度を1秒窓のローリングRMSで計算（50%オーバーラップ）
2. ローリングRMS値の変動係数を計算: `CV = σ(rolling_rms) / μ(rolling_rms)`
3. CVからzooming speedにマッピング:

```
zooming_speed = clamp(6.0 - 4.0 × min(CV, 1.0), 2.0, 6.0)
```

**なぜ変動係数か**: Zooming speedはDynamic zoomingのFOV遷移速度を制御する。ブレが時間的に均一であればFOVをゆっくり変えても問題ないが、ブレが間欠的（急に激しくなったり静かになったりする）であればFOVを素早く追従させる必要がある。CVはこの「時間的な不均一さ」を定量化する指標。

- CV ≈ 0（均一なブレ）→ 6.0s（ゆっくりしたFOV遷移で十分）
- CV ≈ 0.5（中程度の変動）→ 4.0s（Gyroflowデフォルト相当）
- CV ≈ 1.0+（間欠的ブレ）→ 2.0s（素早いFOV追従が必要）

## 制約と注意事項

- 推奨値は信号処理に基づく客観的な推定であり、映像の主観的な品質評価は含まない
- パン・チルト等の意図した動きとブレの区別は周波数帯域の分離に依存しており、非常にゆっくりしたブレ（<0.3Hz）は意図した動きと区別できない
- Gyroflowの「Default」アルゴリズム前提。「Plain 3D」や軸別（Per-axis）モードでは推奨値の解釈が異なる
- 極端に短いクリップ（2秒未満）では周波数分解能が不足し、推奨精度が低下する
