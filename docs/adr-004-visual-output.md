# ADR-004: --visual / --output-image / --sparkline 詳細仕様

## ステータス

改訂済み（2026-03-06）— 初版 2026-03-05 のレイアウト・配色を全面見直し

## コンテキスト

MVP（テキスト出力）は完成しており、v2マイルストーンとしてグラフィカル出力を実装する。初版（2026-03-05）では縦3段スタック（合成角速度時系列＋3軸分解折れ線＋ヒートマップ）のレイアウトを定義したが、以下の問題が判明した:

1. **800×600に3パネルは情報密度が高すぎ**、人間には読み取れない
2. **時系列グラフの横軸がクリップ長に依存**し、短尺・長尺で情報密度が大きく変わる
3. **情報の重複**が多い（AXIS POWERとAXIS ANALYSIS、STATSとSCORE、ヒートマップと波形）
4. 本ツールの価値は実用性よりも**未来的なグラフィックのかっこよさ**にある — デザインはそれを最優先すべき

これらを踏まえ、HUDスタイルのレイアウトに全面改訂する。

## 決定

### CLIオプション

| オプション | 説明 | 単独利用 | 併用 |
|---|---|---|---|
| `--visual` | Sixel/iTerm2でターミナルにグラフ描画 | 可 | `--output-image` と併用可 |
| `--output-image <PATH>` | PNG画像ファイルに書き出し | 可（ターミナル表示なし） | `--visual` と併用可 |
| `--sparkline` | ANSIスパークラインをテキスト出力に追加 | 可 | `--visual` とは排他（`--visual` 優先） |
| `--sixel` | Sixelプロトコルを強制 | — | `--visual` のサブオプション |
| `--iterm2` | iTerm2プロトコルを強制 | — | `--visual` のサブオプション |

使用例:

```bash
gyrotriage clip.MP4 --visual                        # ターミナルにグラフ表示
gyrotriage clip.MP4 --output-image shake.png        # ファイル出力のみ
gyrotriage clip.MP4 --visual --output-image shake.png  # 両方
gyrotriage clip.MP4 --sparkline                     # テキスト+スパークライン
```

### 画像仕様

- **サイズ**: 1440×900px 固定
- **背景**: ダーク（`#1a1b26`）
- **フォーマット**: PNG
- **画像生成**: plottersクレート（またはRust画像生成ライブラリ）
- **ターミナル表示**: `--visual` 時はターミナル横幅一杯に、8:5のアスペクト比を維持してスケーリング表示（iTerm2: `width=100%`、Sixel: `ioctl(TIOCGWINSZ)`でピクセル幅取得しリサイズ）

### レイアウト（2カラム + ヘッダー + フッター）

```
┌─────────────────────────────────────────────────┐
│ GYROTRIAGE v1.0          DJI_xxx.MP4            │ Header
│                          30.3s | 60593 | 2000Hz │
├──────────┬──────────────────────────────────────┤
│ SCORE    │ COMPOSITE (折れ線)                    │
│ ╭──────╮ │──────────────────────────────────────│
││ 72   ││ PITCH (折れ線)                        │
││MODER.││──────────────────────────────────────│
│ ╰──────╯ │ ROLL (折れ線)                        │
│ RMS/PEAK │──────────────────────────────────────│
│ [badge]  │ YAW (折れ線)                         │
├──────────┤                                      │
│ AXIS     │                                      │
│ ANALYSIS │                                      │
│ (レーダー)│                                      │
├──────────┴──────────────────────────────────────┤
│ Recommended Gyroflow parameters: ...            │ Footer
└─────────────────────────────────────────────────┘
```

#### 左カラム（幅: 全体の約1/3）

**SCOREパネル（上半分）:**
- 円弧スコアゲージ（タコメーター型）
  - 弧はスコア位置に応じたグラデーション描画（緑→シアン→青→紫→赤）
  - グロー（にじみ）エフェクト付き
  - 目盛り: 0, 25, 50, 75, 100
- ゲージ中央にスコア数値（大文字）
- レベル名（STABLE/MILD/MODERATE/SEVERE）をスコア色で表示
- RMS値、PEAK値
- MODERATE以上の場合のみ「Gyroflow recommended」バッジ表示

**AXIS ANALYSISパネル（下半分）:**
- 3軸レーダーチャート（Pitch/Roll/Yaw）
- データポリゴンを半透明フィルで描画
- 各頂点にラベルと値（°/s）を表示
- グリッド: 同心三角形（25%/50%/75%/100%）+ 放射線

#### 右カラム（幅: 残り約2/3）

4本の折れ線グラフを縦に等分配置:

1. **COMPOSITE** — 合成角速度（3軸のRSS）
2. **PITCH** — ピッチ軸の角速度
3. **ROLL** — ロール軸の角速度
4. **YAW** — ヨー軸の角速度

各グラフの仕様:
- **ライン描画**（塗りつぶしなし）。グローエフェクト（太い半透明ライン + 細い明るいライン）で視認性を確保
- **横軸**: 固定幅にリサンプル。**時間ラベルなし**（クリップ長が不定のため絶対秒数は表示しない）
- **縦軸**: 相対的な揺れの強度（各グラフ内で正規化）
- **グリッド**: 薄い水平線・垂直線

#### ヘッダー

- 左: ツール名 + バージョン
- 右: ファイル名、Duration/サンプル数/サンプリングレート

#### フッター

- 判定レベルがMODERATE以上（Gyroflow推奨）の場合のみ表示
- 表示内容: `Recommended Gyroflow parameters: smoothness=XX%  max=X.XXXs  max@hv=X.XXXs  zoom_limit=XXX%  zooming_speed=X.Xs`
- STABLE/MILDの場合はフッター行を表示しない

### データのダウンサンプリング

高サンプリングレートデータ（2000Hz等）を固定幅のグラフ領域に描画するため:

1. グラフの有効描画幅（ピクセル数）を算出
2. 全サンプルを描画幅と同数の区間に分割
3. 各区間のRMS角速度を計算してプロット点とする
4. 4グラフ（Composite/Pitch/Roll/Yaw）すべてに同一の手順を適用

### Sixel / iTerm2 プロトコル自動検出

`--visual` 指定時、以下の優先順位でプロトコルを判定する。

1. `--sixel` フラグ → Sixel強制
2. `--iterm2` フラグ → iTerm2強制
3. `TERM_PROGRAM == "WezTerm"` → Sixel
4. `TERM_PROGRAM == "iTerm.app"` → iTerm2
5. `TERM` に `"sixel"` を含む → Sixel
6. いずれにも該当しない → エラー終了（`--sparkline` の使用を案内）

### ターミナル表示サイズ

`--visual` 指定時、ターミナルの横幅一杯に画像を表示する。

- 元画像は1440×900で生成
- iTerm2: `width=100%` でターミナル幅に自動フィット
- Sixel: `ioctl(TIOCGWINSZ)` でターミナルのピクセル幅を取得し、画像が幅を超える場合はLanczos3リサイズ（アスペクト比維持）

### スパークライン（ANSIフォールバック）

`--sparkline` 指定時、通常のテキスト出力の末尾にスパークラインを追加する。

```
File:        DJI_20260228080801_0003_D.MP4
Duration:    30.3s (60593 samples @ 2000Hz)
Score:       100 / 100
Level:       SEVERE
...
---
Gyroflow:    smoothness=21%  max=0.300s  max@hv=0.030s
             zoom_limit=118%  zooming_speed=2.6s

Shake:  ▁▂▃▅▇█▇▅▃▂▁▂▄▆█▇▅▃▁▁▂▃▅▇█▆▄▂▁▁
Pitch:  ▁▁▁▂▂▃▂▂▁▁▁▁▂▂▃▂▁▁▁▁▁▁▂▂▃▂▂▁▁▁
Roll:   ▁▂▃▅▇█▇▅▃▂▁▂▄▆█▇▅▃▁▁▂▃▅▇█▆▄▂▁▁
Yaw:    ▁▁▂▃▃▄▃▃▂▁▁▁▂▃▄▃▃▂▁▁▁▁▂▃▃▄▃▂▁▁
```

- ブロック文字8段階: `▁▂▃▄▅▆▇█`
- 表示幅: ターミナル幅 − ラベル幅（デフォルト60文字、ラベル8文字=データ52文字）
- 値のマッピング: 各区間のRMS角速度を0〜区間最大値で正規化し8段階に量子化
- ANSIカラー: 画像グラフと同じ寒色体系（Composite=シアン/Pitch=青/Roll=ティール/Yaw=紫）

### `--visual` と `--sparkline` の排他

両方指定された場合は `--visual` を優先し、スパークラインは表示しない（グラフィカル出力がスパークラインの上位互換であるため）。

## 依存クレート（追加）

| クレート | 用途 |

| `plotters` | グラフ画像生成（BitMapBackend） |
| `image` | PNG生成・Sixel向けリサイズ |

## 根拠

### 初版（2026-03-05）からの変更理由

- **縦3段→2カラム**: 時系列グラフの横軸がクリップ長に依存する問題を解消。左カラムにスコア・レーダーを固定配置し、右カラムに折れ線を配置することでレイアウトの安定性を確保
- **ヒートマップ廃止**: 4本の折れ線グラフが時間×強度の情報を完全にカバーするため、ヒートマップは劣化版にしかならない
- **AXIS POWER/STATS廃止**: SCOREパネル・AXIS ANALYSISと情報が重複していた。RMS/PEAKはSCOREパネルに統合
- **塗りつぶし→ライン**: 3軸を重ねても視認でき、オシロスコープ的なかっこよさが出る。グローエフェクトとの相性も良い
- **配色を寒色基調に**: 暖色系（赤/黄）はスコアゲージの高スコア域のみに限定。折れ線は寒色4色で統一し、ダーク背景上で映えるSF的な美しさを追求
- **時間ラベル廃止**: クリップ長が不定のため絶対秒数を表示する意味がない。グリッド線のみで相対的な位置感を提供
- **画像サイズ1440×900**: 8:5のアスペクト比。フォントの視認性を確保しつつターミナル横幅一杯に表示した際のバランスが良い
- **フッターの条件付き表示**: Gyroflow推奨でない場合に推奨パラメータを表示する意味がないため
- **かっこよさ最優先**: 本ツールの実用的価値は低い（Gyroflowを無条件に掛ければよい）。未来的なHUDグラフィックがターミナルに表示される体験こそがこのツールの存在意義
